When Will These Tools Be Available?

Firefox is following a "release train" model and these trains run on time. A Firefox release goes out on schedule, but with a set of features that is hard to predict in advance. Our goal is to have most of what you see in the document hit the Aurora release channel in late September (which means beta in November and final release in December).

As I write this, some of these features are already available in the Nightly channel and there will be more landing there regularly over the next few weeks. Even so, it's possible that some features displayed here are not going to ship in the December (Firefox 9) release.

Also, keep in mind that these are mockups and screenshots of in-development tools. I'd be surprised if things don't change between now and the release!

Page Tools vs. Element Tools

Different tools will be available to the user at different times, depending on what the user is doing or has selected. Page tools work with information that is always available on the page (style sheets, scripts, etc.). The Web Console, Scratchpad and Style Editor are all examples of page tools.

There are also element tools which work with either a single page element or a collection of elements. The Style Inspector is an example of an element tool, because it only makes sense to talk about styling in the context of a specific part of the page. Tools users will use a page tool, such as the Highlighter, to select an element to use with an element tool.

The Style Editor could conceivably work at the page level or at the element level.

The Tools

Highlighter

Visually inspecting your page

The Highlighter is the visual entry point into the developer tools. It allows you to quickly zero in on an element of your page and get useful information about the element.

In the example above, the Google logo is highlighted. Highlighter dims the rest of the page, and provides extra information about the highlighted page element in the Infobar. The Highlighter Toolbar appears above the Developer Toolbar at the bottom of the window, providing Highlighter-specific controls.

The Highlighter has two modes: "highlight" and "selected".In "highlight" mode, moving the mouse around will highlight the element of the page that's under the cursor. If you click on the element, you switch to "selected" mode and you can then move your mouse around without changing which element is highlighted. You can click the "Highlight" button in the Highlighter Toolbar to switch back into highlight mode.

Highlighter Infobar

When you highlight a page element, the "infobar" appears to quickly give you the most useful information about that element (tag name, ID of the element and the CSS classes on the element). Additionally, if you click a class name, the Highlighter will toggle that class for that element.

Highlighter Toolbar

The Highlighter Toolbar provides breadcrumb-style navigation to other elements in the document and buttons to toggle other element tools. When you toggle the other element tools, they appear in the sidebar on the right. Only one tool is visible in the sidebar at a time.

Notes and Questions

  • What happens to the breadcrumbs when you have a very deeply nested tree? Is there truncation at one part of the tree or other? (my initial thought is to truncate at the tree's route)

Style Inspector

The Style Inspector provides detailed information about the styles applied to the node, organized by property name. The property names are links that take you to the MDN page that documents that property. For any given property, the selected rule is displayed, along with rules that were overridden. There's also an option to view unmatched rules, which helps in cases where a selector was mistyped.

By default, it only shows the styles that have been altered by user style sheets. There's a checkbox to change this behavior. Also, since there are potentially a very large number of properties to be displayed, there is a filtering search box.

Notes and Questions

  • What will the default sort be? (it's alphabetical by property name at the moment)
  • Can we make the Style Inspector show (in bold, or something like that) which font was selected from a list of fonts?

Style Editor

Layout

The Layout tool displays information about an element's layout. It displays:

  • Position on the page
  • Outer height and width
  • Margins
  • Borders
  • Padding
  • Inner height and width

Notes and Questions

  • Will the layout tool appear in the sidebar or as a doorhanger?
  • Will there be anything special for flexboxes?
